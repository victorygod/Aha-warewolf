# 前端板子信息展示方案

## 现状问题

1. **选板阶段**：板子列表含角色和规则，信息完整，但锁定后只显示"板子已锁定: 12人守丘局"，丢失了角色和规则信息
2. **游戏中**：头部只显示 `12人守丘局 | 第2天`，角色配置和规则描述完全不可见
3. **手机端**：`#app` 最大宽度 600px，头部 sticky，底部操作区 sticky，中间消息区可滚动，空间非常紧凑

## 需要展示的信息

| 信息 | 选板阶段 | 游戏中 |
|------|---------|--------|
| 板子名称 | ✅ 已有 | ✅ 已有（头部） |
| 角色配置（3狼/预言家/女巫/...） | ✅ 已有 | ❌ 缺失 |
| 规则描述（女巫仅首夜可自救/...） | ✅ 已有 | ❌ 缺失 |
| 玩家自己的角色 | — | ✅ 已有 |

## 设计方案：头部点击展开规则面板

### 核心思路

游戏中头部空间有限，不能常驻显示角色配置和规则。改为：**点击头部区域展开一个半屏面板**，展示板子完整信息。这是移动端常见的交互模式（类似微信聊天顶部的群信息入口）。

### 交互细节

1. **头部增加提示箭头**：当有板子信息时，头部右侧显示 `▾` 小图标，暗示可点击
2. **点击头部**：从顶部滑出一个面板（overlay），覆盖消息区域，不覆盖底部操作区
3. **面板内容**：
   - 板子名称（大字）
   - 角色配置（一行：3狼 / 预言家 / 女巫 / 猎人 / 3民）
   - 规则描述（列表）
4. **关闭方式**：再次点击头部 / 点击面板外区域 / 向上滑动关闭
5. **选板阶段锁定后**：同样展示完整信息（角色+规则），不再只显示"板子已锁定"

### 面板布局（ASCII mockup）

```
┌──────────────────────────────┐
│  12人守丘局            ▴ 关闭 │  ← 点击头部或此行关闭
├──────────────────────────────┤
│  角色                         │
│  4狼 / 预言家 / 女巫 / 守卫 / │
│  猎人 / 丘比特 / 3民          │
├──────────────────────────────┤
│  规则                         │
│  · 女巫仅首夜可自救           │
│  · 守卫不可连守               │
│  · 同守同救则死亡             │
│  · 猎人被毒不能开枪           │
│  · 首夜和白天死亡有遗言       │
│  · 情侣一方死亡另一方殉情     │
└──────────────────────────────┘
```

### 手机端适配

- 面板最大高度 40vh，内容超出时内部滚动
- 面板宽度跟随 `#app`（max-width 600px）
- 字体：名称 16px，角色 14px，规则 13px
- 面板背景：`rgba(0, 0, 0, 0.9)`，与现有 `.panel` 风格一致
- 动画：`transform: translateY(-100%)` → `translateY(0)`，300ms ease

### 选板阶段锁定后的展示

当前锁定后只显示"板子已锁定: 12人守丘局"。改为同样展示角色和规则：

```
┌──────────────────────────────┐
│  板子已锁定: 12人守丘局       │
│  4狼 / 预言家 / 女巫 / 守卫 / │
│  猎人 / 丘比特 / 3民          │
│  · 女巫仅首夜可自救           │
│  · 守卫不可连守               │
│  ...                          │
└──────────────────────────────┘
```

锁定后信息直接内联展示，不需要点击展开（此时没有游戏进行，屏幕空间充足）。

## 改动清单

### `public/index.html`

- 头部 `#header` 增加 `cursor: pointer`（CSS 中加）
- 新增 `<div id="preset-panel" class="hidden">` 面板元素，放在 `#app` 内、`#header` 后

### `public/style.css`

- `#header` 加 `cursor: pointer`
- 新增 `#preset-panel` 样式：
  - `position: fixed` / `absolute`，从顶部滑出
  - `max-height: 40vh`，内部 `overflow-y: auto`
  - `z-index: 150`（低于 `.panel` 的 200）
  - 滑入/滑出动画
- 新增面板内部元素样式：`.preset-panel-name`、`.preset-panel-roles`、`.preset-panel-rules`
- `#preset-locked` 扩展：显示角色和规则（不再只是一行文字）

### `public/app.js`

- `updateHeader()`：
  - 有板子信息时，头部右侧加 `▾` 图标
  - 点击头部 toggle `#preset-panel`
  - 面板内容从 `state.preset` 动态渲染
- `showPresetLocked()`：
  - 锁定后显示角色配置和规则描述（内联，非面板）
- 点击面板外区域关闭面板

## 不改动的部分

- 板子选择器（`#preset-list`）：选板阶段的交互不变
- 头部的基本信息（阶段、天数）：不变，面板只是补充
- 底部操作区：不变
