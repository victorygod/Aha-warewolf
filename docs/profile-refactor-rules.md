# AI Profile 改造规则

适用场景：狼人杀桌游 + 聊天室。改造目标：去除与场景无关的信息，保留和提炼对角色扮演有价值的内容。

## background.md 规则

### 删除
- 游戏版本号前缀（1.0、1.3、2.4、3.0、3.6、3.8等）
- 纯游戏机制数据：星级、属性、命途/命途类型（存护/巡猎/记忆等）、战技/终结技/普攻的具体机制描述
- 抽卡/实装信息（常驻/限定/UP次数）
- 开发元信息（字数统计、配音评价、PV标题等）
- 纯形态切换事件（如"获得巡猎形态"），除非事件本身含有人际关系或情感内容

### 保留
- 角色名称、命名由来
- 身世背景和核心谜团（失忆、身世之谜等）
- 与其他角色的关系和共同经历
- 社区梗/出圈梗（其他角色调侃时可能引用）
- 外貌特征

### 改造
- 游戏机制描述 → 提炼为性格本能。例："战技单体套盾"→"优先保护队友"；"记忆命途，召唤忆灵长夜"→"拥有与记忆/回忆相关的力量"
- 剧情事件 → 去掉版本号和任务名，保留事件本身和人际互动。例："1.3：同行任务全面回忆，借穷观阵探索被封印记忆"→"曾借助穷观阵探索被封印的记忆"
- 过于具体的剧情节点 → 凝练为可聊天的共同回忆。例："协助缇里西庇俄斯出逃"→"帮助过朋友逃离困境"

## thinking.md 规则

### 删除
- 末尾的"XX型"总结段落（与前面内容重复的归纳标签）
- 纯战斗场景的思维逻辑（如"战斗中优先为队友提供护盾"）
- 游戏技能名作为思维模式标题（如"冰冻记忆"）

### 合并
- 同一逻辑的重复表达合并为一条。常见重复模式：原则陈述 → 行为表现 → 型格标签，三者只保留最精炼的一条
- 语义相近的多条合并。例："害怕孤独"和"不独自承受困难"→"害怕孤独，遇到困难一定找朋友帮忙"

### 改造
- 战斗场景思维 → 映射为通用场景。例："战斗中优先保护队友"→"危机中优先保护同伴"
- 游戏技能梗 → 换表达保留本质。例："冰冻记忆：有些事先放一放"→"暂时搁置：有些事先放一放，不急于求解"
- 口头禅/行为习惯 → 迁移到speaking.md

### 保留
- 核心人生观和价值观
- 面对困境/未知/质疑的反应模式
- 追求和拒绝的事物
- 性格特质描述

## speaking.md 规则

### 删除
- 游戏术语标签：战技名、普攻语音、PV标题、角色实装标签等
- 开发元信息（字数统计、配音评价等）
- 玩家社区评价（角色不会这样评价自己）

### 改造
- "战斗"场景标签 → "游戏/紧张时刻"场景标签
- 游戏技能名作为口头禅 → 保留口号，去掉来源标注。例："可爱即正义：战技名兼代表口号"→"可爱即正义"
- 游戏内行为 → 映射为桌游/聊天场景。例："拍照留念"→在狼人杀中体现为"记住关键时刻的细节"

### 保留
- 自称和语气词
- 句式特征和语速
- 各社交场景的说话方式（初见/好友/前辈/困境等）
- 口头禅和代表语录
- 社区梗/出圈梗（其他角色可以引用来调侃）
